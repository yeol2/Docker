# 🚀 엔터프라이즈 서버 운영환경과 도커  

기업에서 사용하기 위한 엔터프라이즈 서버 운영환경은 크게 **3가지 방식**이 있다.  

## 🏗 1. 베어메탈 (Bare Metal)  
- 물리 서버에서 운영체제(OS)를 직접 설치하고 운영하는 방식  
- **특징:**  
  - 성능이 뛰어나지만, 리소스 활용도가 낮음  
  - 확장성과 유연성이 떨어짐  
  - 하드웨어 의존성이 큼  
- **사용 사례:** 대규모 데이터베이스 서버, 고성능 컴퓨팅(HP) 환경  

---

## 🖥 2. 하이퍼바이저 (Hypervisor)  
- 물리 서버 위에서 **가상화(Virtualization)** 기술을 활용하여 다수의 VM(가상 머신)을 운영하는 방식  
- **특징:**  
  - 여러 개의 운영체제를 동시에 실행 가능  
  - 물리 자원을 효율적으로 분할하여 사용 가능  
  - 가상화 소프트웨어(예: VMware, KVM, Hyper-V)를 사용  
- **사용 사례:** 클라우드 서버(AWS, Azure), 개발/테스트 환경  

---

## 📦 3. 컨테이너 (Container)  
- OS 위에서 컨테이너 엔진(예: **Docker, Kubernetes**)을 활용하여 애플리케이션을 격리된 환경에서 실행하는 방식  
- **특징:**  
  - 가볍고 빠름 (VM보다 리소스를 적게 사용)  
  - 운영체제 커널을 공유하여 효율적  
  - 애플리케이션의 이식성과 확장성이 뛰어남  
- **사용 사례:** 마이크로서비스 아키텍처(MSA), DevOps, CI/CD 환경  

---

## 🎯 우리가 배울 것은 **컨테이너 (Docker)** 🚢  
도커(Docker)는 컨테이너 기술을 활용한 애플리케이션 배포 및 관리 도구이다.  
컨테이너 방식은 기존의 VM 방식보다 **더 가볍고 빠른** 배포가 가능하여,  
많은 기업에서 **DevOps, 클라우드, 마이크로서비스** 환경에서 널리 사용된다.  

### 🔹 **도커를 사용하면?**
✅ 빠른 애플리케이션 배포 및 실행  
✅ 가벼운 환경 제공 (하이퍼바이저보다 리소스 절약)  
✅ 환경 차이에 영향을 받지 않는 이식성  

🚀 앞으로 도커를 배워서 실제 환경에서 어떻게 활용되는지 익혀보자!
